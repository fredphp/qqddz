# 斗地主游戏匹配流程设计方案

## 一、问题分析

当前匹配系统存在的问题：
1. **快速匹配需要等待3人**：玩家点击快速匹配后，如果队列人数不足3人，会一直等待
2. **没有等待状态反馈**：玩家不知道当前有多少人在匹配
3. **无法主动创建房间等待**：玩家只能被动等待匹配
4. **游戏场景无等待UI**：进入游戏场景后，如果人数不足没有明确提示

### 2.3 技术实现要点

#### 2.3.1 客户端改动

1. **hallScene.js 修改**
   - `_quickMatch()` 改为智能匹配：
     1. 先请求获取可加入房间列表
     2. 如果有等待中的房间，加入该房间
     3. 如果没有，创建新房间

2. **gameScene.js 修改**
   - 添加等待模式UI
   - 监听 `player_joinroom_notify` 事件更新玩家列表
   - 房间满员时自动准备并开始游戏

3. **socket_ctr.js 改进**
   - 新增 `getWaitingRooms()` 获取等待房间
   - 新增 `cancelMatch()` 取消匹配

#### 2.3.2 服务器端改动（已完成）

服务器端已支持：
- `quick_match`: 快速匹配（队列模式）
- `create_room`: 创建房间
- `join_room`: 加入房间
- `get_room_list`: 获取房间列表

### 2.4 用户体验优化

1. **匹配进度提示**
   - 显示当前匹配队列人数
   - 显示预计等待时间
   - 动态加载动画

2. **房间状态同步**
   - 实时显示房间人数变化
   - 玩家加入/离开通知
   - 准备状态显示

3. **超时处理**
   - 等待超过60秒提示是否继续等待
   - 提供离开房间选项

## 三、实现计划

### Phase 1: 智能匹配逻辑
- 修改快速匹配为智能匹配
- 优先加入现有等待房间

### Phase 2: 等待界面
- 创建等待中UI组件
- 显示房间状态和玩家列表

### Phase 3: 状态同步
- 实现房间状态实时更新
- 添加玩家加入/离开通知

### Phase 4: 体验优化
- 添加匹配进度动画
- 添加超时提示
- 添加邀请好友功能

## 四、API 设计

### 4.1 客户端 -> 服务器

```javascript
// 智能匹配（优先加入现有房间）
socket.emit('smart_match', { room_level: 2 });

// 获取等待房间列表
socket.emit('get_waiting_rooms', { room_level: 2 });

// 创建房间并等待
socket.emit('create_room', {});

// 加入指定房间
socket.emit('join_room', { room_code: 'ABC123' });

// 离开房间
socket.emit('leave_room', {});

// 取消匹配
socket.emit('cancel_match', {});
```

### 4.2 服务器 -> 客户端

```javascript
// 房间加入成功
{ type: 'room_joined', room_code: 'ABC123', players: [...] }

// 玩家加入通知
{ type: 'player_joined', player: { id, name, seat } }

// 玩家离开通知
{ type: 'player_left', player_id: 'xxx' }

// 游戏开始
{ type: 'game_start', room_code: 'ABC123' }

// 匹配状态更新
{ type: 'match_status', queue_count: 2, estimated_time: 30 }
```
